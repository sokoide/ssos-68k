# SSOS3 アーキテクチャ概説

SSOS3 は X68000 実機およびエミュレータ上で動作する、ベアメタル OS プロトタイプです。

## グラフィックモード設定

## VRAM 構造の違い

X68000 には、色数に応じて「4プレーン形式」と「パックド形式」の2種類の VRAM 管理方式があります。

### 1. 4プレーン形式 (Bitplane Structure)
16色モードで使用される伝統的な形式です。1つの画素を表現する 4ビットの情報が、4つの独立したメモリ平面（プレーン0〜3）に分散して格納されます。
- **データ配置**: メモリ上の同じアドレスにある 4つのプレーンの特定のビットを重ね合わせることで、0〜15 の色インデックスを生成します。
- **特徴**: 特定のプレーン（色成分）だけを一括で書き換える処理に向いていますが、1画素の色を直接指定するには各プレーンへの書き込みが必要なため、計算が複雑になります。
- **アドレス**: `$C00000`（プレーン0）, `$C40000`（プレーン1）, `$C80000`（プレーン2）, `$CC0000`（プレーン3）と、プレーンごとに広いアドレス空間が割り当てられます。

### 2. パックド形式 (Packed-Pixel Structure)
256色および 65536色モードで使用される、現代的な GPU に近い形式です。
- **データ配置**: 1つの画素の情報（8ビットまたは16ビット）が、メモリ上の連続したアドレスにそのまま格納されます。
  - **256色モード**: 1画素 = 1バイト (8ビット)
  - **65536色モード**: 1画素 = 1ワード (16ビット)
- **特徴**: アドレス計算が `y * line_width + x` と非常に単純で、C言語などの高級言語から直感的にピクセルを操作するのに適しています。
- **アドレス**: `$C00000` から連続したリニアな空間として扱えます。

## VRAM 構成と座標系 (Mode 8 採用時)

SSOS3 で使用しているパックド形式（65536色）の構成です。

- **GVRAM アドレス**: `$C00000`
- **1画素**: 16ビット (High Color / R:5, G:5, B:5 + Priority:1)
- **ライン幅 (Stride)**: 常に **512 画素分（1024 バイト）**。
- **座標計算**: `GVRAM + (y * 512) + x`

## フォント描画アーキテクチャ

X68000 の CGROM（文字パターン ROM）からフォントデータを取得し、GVRAM へ直接描画します。

### CGROM メモリマッピング
実装の簡潔さを優先し、以下の「パック形式」を採用しています。
- **8x8 ANK フォント (パック形式)**: `$F3A000`
  - 1文字あたり 8バイト（8行 × 1バイト）。
  - ベースアドレス: `CGROM + 0x3A000`
  - 文字 `c` の開始アドレス: `ベースアドレス + (c * 8)`

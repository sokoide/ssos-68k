# SSOS3 アーキテクチャ概説

SSOS3 は X68000 実機およびエミュレータ上で動作する、ベアメタル OS プロトタイプです。

## フォント描画アーキテクチャ

X68000 の CGROM（文字パターン ROM）からフォントデータを取得し、GVRAM（グラフィックス VRAM）へ直接描画します。

### 1. CGROM メモリマッピング

X68000 の CGROM（`$F00000` - `$FBFFFF`）には、同一のフォントデータが複数のレイアウトでマッピングされています。SSOS3 では実装の簡潔さを優先し、以下の「パック形式」を採用しています。

#### パック形式 (Packed Layout)
1行のドットデータ（8ビット）が1バイトに格納され、メモリ上に連続して配置されている形式です。
- **8x8 ANK フォント (パック形式)**: `$F3A000`
  - 1文字あたり 8バイト（8行 × 1バイト）。
  - ベースアドレス: `CGROM + 0x3A000`
  - 文字 `c` の開始アドレス: `ベースアドレス + (c * 8)`
  - **メリット**: 1文字が 8バイト の連続ブロックであるため、ポインタを単純にインクリメントするだけで1行ごとのデータが取得でき、ループ処理が極めて単純になります。

#### ワードアライメント形式 (Word-Aligned Layout)
16ビットバスの特性に合わせ、1行（8ビット）を1ワード（16ビット）の上位バイトに配置し、下位バイトを空き（またはダミー）とする形式です。
- **8x8 ANK フォント (ワード形式)**: `$F32000`
  - 1文字あたり 16バイト。
  - **補足**: IOCS の `_fntget` などはこの形式を前提としていますが、ベアメタル描画ではパック形式の方がコードが簡潔になります。

### 2. 描画アルゴリズム (`draw_char8`)

`ssos3/standalone/main.c` における描画フローは以下の通りです。

1.  **正規化**: 文字コードが 0x20-0x7E（英数字）および 0xA1-0xDF（半角カナ）の範囲外であればスペースに置き換えます。
2.  **アドレス算出**: `0xF3A000 + (c * 8)` により対象文字の先頭アドレスを特定します。
3.  **ビット展開描画**:
    - 文字の高さ（8行）分ループ。
    - 各行のバイトデータから各ビットを検査。
    - ビットが 1 であれば前景色、0 であれば背景色を 16ビット（65536色モード）のピクセル値として GVRAM へ書き込みます。

### 3. VRAM 構成 (High Color モード)

SSOS3 は 65536色モード（CRTMOD 13 / 320x256画素）を使用します。
- **GVRAM アドレス**: `$C00000`
- **1画素**: 16ビット (R:5, G:6, B:5)
- **ライン幅**: 内部的に 512 画素分（1024 バイト）が確保されています。
- **座標計算**: `GVRAM + (y * 512) + x`

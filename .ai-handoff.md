### 確定した真因
IOCS `_B_PRINT` ハンドラが **a1 を scratch（文字列カーソル/ワークポインタ）に使い、
戻り時に復元しない**。後続の `_B_SUPER(0x81)` が同じ trap#15 ABI で **MD を a1 から読み**、
a1=BSS+1（IOCS が残したワークアドレス）を MD と誤認 → user 復帰で **USP=BSS+1** →
`_exit` の DOS 終了処理が壊れたスタックを歩き IPL-ROM の逆方向 memmove (0xFFA870,
`move.l -(a0),-(a1)`) で Address Error。
- cooperative が無傷だったのは BSS 配置の差で a1 残留値が偶然安全値だったため（**Timer D /
  プリエンプションは無関係**。当初の Timer D 仮説は誤り。CASE 11 で IPL=7 が OK だったのも
  B_PRINT 後の a1 がたまたま安全値だった非決定的結果）。
- 31バイト文字列で安定再現、12バイトでは再現しなかったのも非決定的（短いと a1 残留値が
  安全側に振れる）。

### 修正（commit 参照）
- **enter**: `a1=0` (MD=0) で supervisor 入り、d0(USP) を `old_usp` に保存。
  MD<0 は IOCS が SSP を MD から導出し起動 crash（a1=-1 で実証: SSP=00008356 奇数破壊）。
- **exit (normal/trap14)**: trap 直前に `a1=old_usp` を明示設定、B_PRINT の a1 破壊を上書き。
- 変数名 `old_ssp` → `old_usp`（`_B_SUPER` 戻り値は USP）。
- IPL=7 保護は削除（Timer D 無関係のため不要）。

### IOCS trap#15 ABI の教訓（今後の指針）
1. 全 IOCS 関数は**第1引数を a1**、関数番号を d0 で渡す（`b_print.S`/`b_super.S` で確認）。
2. IOCS ハンドラは a1 を scratch に使い**戻り時に復元しない**（B_PRINT で実証）。
3. 生 trap で IOCS を連続呼び出しする場合、**各 trap の直前に a1 を明示設定**が必須。
   C ラッパ（`_iocs_b_print` 等）は内部で a1 を設定するが、呼出後の a1 は保証されない。
4. `_B_SUPER` の MD: **0 = supervisor入り**、**>0 = user戻り(USP=MD)**、**<0 は不可**
   （SSP を MD から導出しスタック破壊）。戻り値 d0 = user時の A7（= USP）。

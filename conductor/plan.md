# ウィンドウドラッグと描画パフォーマンスの改善

## 1. ドラッグ終了時の枠線残留の修正
ドラッグ終了時（マウスボタン離上時）に、VRAMに直接描画されたXOR枠線を消去する処理を追加します。
現在は `ssosmain.c` のメインループで `g_drag.is_dragging` が `true` の時だけXOR枠線を描画していますが、`false` になった直後の1フレームで消去（同じ座標に再描画）が行われないため、点線が残る可能性があります。

### 修正内容
`ss_drag_update_frame` 関数内でドラッグ終了を検知した際、直前の座標でXOR描画をもう一度行い、VRAM上の枠線を消去します。

## 2. 描画パフォーマンスの向上 (768x512対応)
ssos2よりも解像度が高いため、ドラッグ中の描画負荷が高いのは事実です。
現状、ドラッグ中も `update_layer_3(l3)` (マウス座標情報のテキスト更新) を行い、その結果発生するダーティリージョンの背景再描画が走っています。これが重い原因の一つです。

### 改善案
- **ドラッグ中の背景再描画の制限**: ドラッグ枠線（XOR）の「跡地」の修復は必須ですが、ウィンドウ内のテキスト更新などの重い処理をドラッグ中のみスキップ、または頻度を落とすことを検討します。
- **ダーティリージョンの最小化**: `ss_drag_update_frame` で枠線移動時に追加しているダメージ矩形のマージンを最適化します。
- **高速なXOR描画**: `ss_xor_dotted_rect_vram` の実装を、ループ内での条件分岐を減らすなどして最適化します。

## 3. 実装ステップ
1. `ssos/os/main/ssoswindows.c` の `ss_drag_update_frame` を修正し、ドラッグ終了時にVRAM上の枠線を消去。
2. `ssos/os/main/ssosmain.c` のメインループを調整し、ドラッグ中の不要なレイヤー更新を最小限に。
3. `ssos/os/kernel/vram.c` の `ss_xor_dotted_rect_vram` を、ワードアクセスを活用した高速版に差し替え検討。
